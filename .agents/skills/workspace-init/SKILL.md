# 工作区初始化专家

你负责初始化新的工作区，使其纳入中心 TODO 管理体系。

---

## 触发条件

当用户要求：
- "初始化工作区"
- "创建工作目录"
- "把这个目录加入管理"
- "注册新工作区"

---

## 初始化流程

### 1. 确认信息

向用户确认：
- **目录路径**: 工作区的完整路径
- **项目名称**: 用于 TODO.md 标题和索引描述
- **目录状态**: 空目录 / 已有内容的目录

### 2. 创建 TODO.md

在工作区根目录创建 `TODO.md`，按规范格式：

```markdown
# TODO - [项目名称]

> 最后更新: YYYY-MM-DD

<!-- 标签: #bug #feature #refactor #test #security #performance #devops #docs -->

## 🔴 进行中


## 🟡 待办


## 🟢 已完成

```

如果目录已有内容，分析项目并生成初始任务：
- 识别技术栈和框架
- 扫描代码质量问题
- 检查配置和安全问题
- 梳理缺失的功能和文档
- 将发现转化为结构化的 TODO 任务

### 3. 创建 AGENTS.md

在工作区根目录创建 `AGENTS.md`，包含：

```markdown
# 工作区 Agent 指令

## TODO 管理规范

本工作区的 Agent 必须遵循中心 TODO 管理规范：

📄 **规范文件**: E:\TODO\TODO_SPEC.md
📄 **全局 Skill**: todo-manager（自动触发）

### 职责要求

1. **读取规范**: 管理 TODO 前先读取 TODO_SPEC.md
2. **遵循格式**: 严格按照规范维护本工作区的 TODO.md
3. **即时更新**: 完成任务后立即更新状态
4. **保持一致**: 确保 TODO 内容与实际情况一致
5. **主动汇报**: 被询问时清晰汇报任务进度

### 快速参考

**状态分类**：
- 🔴 进行中（最多 3 项）
- 🟡 待办
- 🟢 已完成（保留 7-30 天后归档）

**任务格式**：
```markdown
- [ ] 任务描述 `P0-P3` `YYYY-MM-DD` `#标签`
```

**优先级**：
- `P0` 紧急且重要（当天完成）
- `P1` 重要不紧急（本周内）
- `P2` 一般任务（本月内）
- `P3` 低优先级/可选

**标准标签**：
`#bug` `#feature` `#refactor` `#docs` `#test` `#devops` `#security` `#performance`

### TODO 文件位置

- **本工作区 TODO**: TODO.md（位于工作区根目录）
- **中心规范**: E:\TODO\TODO_SPEC.md
- **工作区索引**: E:\TODO\WORKSPACES.md
```

### 4. 项目分析（如果有内容）

如果目录已有代码/文档，执行以下分析：

#### 技术栈识别
- 编程语言和版本
- 框架和库
- 构建工具和包管理器
- 配置文件（package.json, pom.xml, requirements.txt 等）

#### 目录结构分析
- 源码目录组织
- 测试目录
- 文档目录
- 配置目录

#### 现有文档梳理
- README.md
- CHANGELOG.md
- API 文档
- 部署文档

#### 问题发现
- 代码质量问题（重复代码、复杂度、命名不规范）
- 安全问题（硬编码密码、SQL 注入风险、XSS 风险）
- 配置问题（环境变量缺失、配置不完善）
- 测试覆盖（缺失测试、测试质量）
- 文档缺失（README、API 文档、部署文档）
- 依赖问题（过时版本、安全漏洞）

将发现的问题转化为 TODO 任务，按优先级和标签分类。

### 5. 注册到中心索引

更新 `E:\TODO\WORKSPACES.md`，在"工作区列表"表格中添加新行：

```markdown
| [项目名称] | [完整路径]/TODO.md | [简短描述] | ✅ 已对齐 |
```

### 6. 完成确认

初始化完成后，向用户汇报：
- 创建的文件列表
- 项目分析结果（如果有）
- 生成的 TODO 任务统计
- 索引注册状态

---

## 示例输出

初始化完成后，工作区应包含：

```
[工作区目录]/
├── TODO.md          # 标准格式的待办清单
├── AGENTS.md        # 本地 Agent 指令
└── [其他项目文件...]
```

中心工作区的 `WORKSPACES.md` 应新增该工作区的索引记录。

---

## 注意事项

1. **TODO.md 必须严格遵循规范**：三状态区、任务格式、标签系统
2. **AGENTS.md 必须引用中心规范**：确保本地 Agent 知道如何管理 TODO
3. **必须注册到索引**：否则中心工作区无法发现这个工作区
4. **分析要实用**：生成的 TODO 任务应该是可执行的，不要过度理想化
5. **优先级合理**：不要把所有任务都标为 P0/P1
