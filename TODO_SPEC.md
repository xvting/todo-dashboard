# TODO 管理规范 v1.0

> 本规范定义各工作区 TODO 文件的标准格式与管理规则，确保跨工作区的一致性和可维护性。

---

## 1. 文件命名与位置

- **文件名**: `TODO.md`（固定，便于自动发现）
- **位置**: 工作区根目录
- **编码**: UTF-8
- **换行符**: LF（Unix 风格）

---

## 2. 状态分类

使用以下三个标准状态区（按优先级排序）：

```markdown
## 🔴 进行中

## 🟡 待办

## 🟢 已完成
```

**状态定义**：
- **进行中**: 当前正在处理的任务（最多 3 项，避免并行过多）
- **待办**: 已规划但未开始的任务
- **已完成**: 已完成并验证的任务（定期归档，建议保留 30 天）

---

## 3. 任务格式

### 3.1 基本格式

```markdown
- [ ] 任务描述
- [x] ~~已完成任务描述~~ `完成日期`
```

### 3.2 完整格式（推荐）

```markdown
- [ ] 任务描述 `优先级` `截止日期` `标签`
```

**字段说明**：
| 字段 | 必填 | 格式 | 示例 |
|---|---|---|---|
| 任务描述 | ✅ | 清晰、可执行的描述 | 实现用户登录功能 |
| 优先级 | ⚠️ | `P0` / `P1` / `P2` / `P3` | `P1` |
| 截止日期 | ⚠️ | `YYYY-MM-DD` | `2026-07-25` |
| 标签 | ⚠️ | `#标签名` | `#bug` `#feature` `#refactor` |

**优先级定义**：
| 级别 | 含义 | 处理时限 |
|---|---|---|
| `P0` | 紧急且重要 | 立即处理，当天完成 |
| `P1` | 重要不紧急 | 本周内 |
| `P2` | 一般任务 | 本月内 |
| `P3` | 低优先级/可选 | 无明确时限 |

### 3.3 分组规则

任务过多时，使用三级标题分组：

```markdown
### 功能开发
- [ ] 任务1
- [ ] 任务2

### Bug 修复
- [ ] 任务3

### 文档
- [ ] 任务4
```

## 4. 任务描述规范

### 4.1 描述原则

- ✅ **可执行**: 使用动词开头，明确要做什么
- ✅ **可验证**: 完成后能明确判断是否达标
- ✅ **原子性**: 一个任务一个交付物，避免过大
- ❌ **避免模糊**: "优化代码" → "重构 UserService 的异常处理逻辑"

### 4.2 描述示例

| ❌ 模糊 | ✅ 清晰 |
|---|---|
| 修复登录问题 | 修复 JWT token 过期后未自动刷新的问题 `#bug` `P0` |
| 添加新功能 | 实现用户密码修改功能（前端表单 + 后端 API）`#feature` `P1` `2026-07-25` |
| 优化性能 | 优化 SQL N+1 查询问题（目标：51 条 SQL → 5 条以内）`#refactor` `P2` |

---

## 5. 文件头部元数据

每个 TODO 文件应包含头部信息：

```markdown
# TODO - [工作区名称]

> 最后更新: YYYY-MM-DD
> 维护者: [姓名/团队]
> 关联文档: [可选，链接到项目文档]
```

---

## 6. 更新规则

### 6.1 日常更新

- **任务状态变更**: 立即更新（开始做 → 移到"进行中"；完成 → 移到"已完成"）
- **新增任务**: 添加到"待办"区，按优先级排序
- **完成任务**: 勾选后保留 7-30 天，然后归档或删除

### 6.2 定期审查

- **每日**: 检查"进行中"任务，确保不超过 3 项
- **每周**: 审查"待办"列表，调整优先级，删除过时任务
- **每月**: 归档"已完成"任务，清理超过 30 天的记录

### 6.3 状态同步

当任务涉及多个工作区时：
- 在主工作区记录核心任务
- 在依赖工作区记录子任务，并标注关联（如：`关联: 主工作区 #任务ID`）

## 7. 标签系统

### 7.1 标准标签

| 标签 | 用途 |
|---|---|
| `#bug` | Bug 修复 |
| `#feature` | 新功能 |
| `#refactor` | 重构/优化 |
| `#docs` | 文档 |
| `#test` | 测试相关 |
| `#devops` | 部署/运维 |
| `#security` | 安全相关 |
| `#performance` | 性能优化 |

### 7.2 自定义标签

工作区可自定义标签，但需在本文件顶部声明：

```markdown
<!-- 标签: #frontend #backend #database #api -->
```

---

## 8. Agent 职责

各工作区的 Agent 应：

1. **遵循格式**: 严格按照本规范维护 TODO 文件
2. **主动更新**: 完成任务后立即更新状态
3. **保持一致**: 确保文件内容与实际情况一致
4. **定期汇报**: 当被询问时，能清晰汇报任务进度
5. **冲突处理**: 发现任务描述不清或冲突时，主动向用户确认
